## Oracle RAC 26ai Ansible

**Repository:** [https://github.com/undrakhorgil/ansible-db](https://github.com/undrakhorgil/ansible-db)

Step-based Ansible automation for a 2-node (or N-node) Oracle RAC lab: OS prep → GI/CRS → ASM → DB software → DBCA.

### Quickstart

```bash
# 1) Install collections
ansible-galaxy collection install -r collections/requirements.yml

# 2) Edit configuration
${EDITOR:-vi} group_vars/all.yml

# 3) Run everything
ansible-playbook playbooks/site.yml
```

### Project structure

```text
ansible-db/
  ansible.cfg                  # Output/log defaults + callback config
  oradbctl.sh                   # Step/undo runner (bash)
  inventory.yml                 # Base inventory (dynamic hosts added at runtime)
  group_vars/
    all.yml                     # Main config (topology, paths, ASM, DBCA)
    vault.yml.example           # Example secrets file (encrypt with ansible-vault)
  collections/
    requirements.yml            # Ansible collections to install
  playbooks/
    site.yml                    # Full run (steps 1–22)
    run_step.yml                # Run one step: -e rac_step=...
    run_undo_step.yml           # Run one undo step: -e rac_undo_step=...
    prep_remote.yml             # Ensure remote python + basic prep
    bootstrap_rac_inventory.yml # Build oracle_rac_servers from rac_nodes
  roles/
    oracle_rac/
      tasks/                    # Step tasks 01–22 + undo_* + helpers
      templates/                # Jinja templates (summary HTML, limits)
  callback_plugins/
    clean.py                    # Cleaner TASK output (no ***** banners)
  rac_environment_summary.html  # Generated by step 21 (control host)
```

### Install dependencies (control host)

```bash
ansible-galaxy collection install -r collections/requirements.yml
```

### Configure

- **Inventory**: `inventory.yml` (hosts are built dynamically from `rac_nodes` in `group_vars/all.yml`)
- **Variables**: `group_vars/all.yml` (cluster topology, paths, ASM disks, DBCA settings)

#### Key variables to review first

- **RAC topology**: `rac_nodes`, `rac_primary_node`, `rac_scan_name`, `rac_scan_ips`
- **Paths/users**: `oracle_base`, `grid_home`, `oracle_home`, `grid_user`, `oracle_user`
- **ASM stable naming**: `asm_disks`, `asm_symlinks` (by-path + `ID_PATH`)
- **Disk groups**: `asm_diskgroups` (`used_for_grid_crs: true` must be exactly one entry)
- **DBCA**: `dbca_*` (especially `dbca_data_diskgroup`, `dbca_recovery_diskgroup`)

#### Secrets (recommended)

Store passwords in Ansible Vault instead of committing them in plaintext:

```bash
cp group_vars/vault.yml.example group_vars/vault.yml
ansible-vault encrypt group_vars/vault.yml
```

### Run (full)

```bash
ansible-playbook playbooks/site.yml
```

### Run (step / undo)

Common step runner usage:

```bash
bash oradbctl.sh -l
bash oradbctl.sh '-?'     # zsh: quote -?
bash oradbctl.sh -s 1
bash oradbctl.sh -s -r 1-22
bash oradbctl.sh -u 9
bash oradbctl.sh -u -r 6-12   # reverse order
```

### Logging / output readability

Default output is tuned in `ansible.cfg`:

- **Less visual noise**: `stdout_callback = clean` (removes long `*****` banners)
- **Structured results**: `[callback_default] result_format = yaml`
- **Timing**: `callbacks_enabled = profile_tasks,timer`

To log to a file:

```bash
ANSIBLE_LOG_PATH=ansible.log ansible-playbook playbooks/site.yml
```

### SSH host key checking

By default the playbooks use a safer SSH behavior (no forced `StrictHostKeyChecking=no`). To keep old lab behavior, set:

```yaml
oracle_ssh_strict_host_key_checking: false
```

### Step map

Use `bash oradbctl.sh -l` to see the same list from the script.

| Step | Name | Runs on | Purpose / notes |
|---:|---|---|---|
| 01 | Precheck networking | all nodes | `/etc/hosts` + basic name resolution checks |
| 02 | OS packages & time sync | all nodes | packages, chrony, SELinux/firewalld lab tweaks |
| 03 | Users, groups, limits | all nodes | `grid`/`oracle` users + groups + PAM limits |
| 04 | Create Oracle directories | all nodes | create empty `ORACLE_BASE`, `GRID_HOME`, `ORACLE_HOME`, inventory |
| 05 | Sync hosts file | all nodes | re-render `/etc/hosts` from `rac_nodes` |
| 06 | Shared disks (ASM) + udev | all nodes | partition + udev rules (supports `/dev/disk/by-path` + `ID_PATH`) |
| 07 | User env & SSH | all nodes | SSH config/known_hosts for CVU + installers |
| 08 | Unzip Grid Infrastructure | primary | unzip GI ZIP into `GRID_HOME` |
| 09 | Run CVU precheck | primary | `runcluvfy.sh stage -pre crsinst` |
| 10 | Grid setup (silent) | primary | `gridSetup.sh` (CRS config stage) |
| 11 | Run root scripts | all nodes | GI `root.sh` |
| 12 | Execute config tools | primary | `gridSetup.sh -executeConfigTools` |
| 13 | Grid postcheck (discover) | primary | locate CRS tools / effective GI home |
| 14 | Grid postcheck (run) | primary | runtime checks (`crsctl`, `olsnodes`) |
| 15 | Unzip DB software | primary | unzip DB ZIP into `ORACLE_HOME` |
| 16 | Install DB software | primary | software-only install |
| 17 | Run DB root scripts | all nodes | DB `root.sh` |
| 18 | DB postcheck | primary | verify DB home / tooling |
| 19 | Create ASM disk groups | all nodes | create non-CRS DGs on primary; mount all DGs on every `+ASMn` |
| 20 | Create database (DBCA) | primary | DBCA RAC create; mounts DGs before DBCA; registers DG deps after success |
| 21 | Summary | control host | writes `rac_environment_summary.html` into the repo |
| 22 | Rotate passwords | all nodes | change OS passwords (root/oracle/grid) |

### Notes

- **Host scope**: steps **1–7** run on every host in `oracle_rac_servers`. Most install steps run primarily on `rac_primary_node` (check each task file’s `when:`).
- **ASM disks**: this repo supports stable naming via `/dev/disk/by-path/*` + udev `ID_PATH` matching (see step 6).
- **Summary output**: step 21 writes `rac_environment_summary.html` on the **control host** (this repo), not on the RAC nodes.

**Step 8 (GI):** Unzips Grid Infrastructure only on **`rac_primary_node`** (see `rac_primary_node` in `group_vars/all.yml`, default `host01`). Other nodes do **not** receive the GI tree from Ansible; copy or sync `GRID_HOME` there if your workflow needs it. Step **8** ensures `{{ grid_home }}` exists before `unarchive` (required by Ansible and useful if step **4** was skipped or paths were removed).

**PAM limits:** step **3** writes `/etc/security/limits.d/99-oracle-rac.conf` and mirrors into `/etc/security/limits.conf` for `grid` and `oracle`.

**Undo (Grid):** **`undo 10`** runs **`$GRID_HOME/deinstall/deinstall`** (Oracle deinstall), then **`cleanup_grid_install_state.yml`** (strip `GRID_HOME` from central `inventory.xml` where present, clear **`/etc/oracle`** locators, **`olr.loc`**, **`scls_scr`**, **`chkconfig`**, and invalid **`oraInst.loc`**). **`undo 8`** and **`undo 4`** run the same cleanup so reinstall is not blocked by stale metadata. **`undo 8`** also clears **contents** of `GRID_HOME` (the empty directory remains). **`undo 7`** does not delete `/u01` Oracle paths.

### Troubleshooting (common)

- **`/dev/oracleasm/asm-*` missing**: ensure `/dev/oracleasm` exists (tmpfs) and udev rule matches `ID_PATH` (step 6).
- **Diskgroup online only on primary**: mount the DG on every ASM instance (`+ASM1..+ASMn`) (step 19 mounts per DG on all nodes).
- **DBCA fails with ORA-15001 (diskgroup not mounted)**: ensure ASM diskgroups are mounted on every node before DBCA (step 20 mounts), and the DB resource has diskgroup dependencies (`srvctl modify database -diskgroup` is done after success).
